1
Create two Discord webhooks

You need two webhooks — one per channel. Do this for both #app-feedback and #card-design:

  1. Open the channel in Discord → click the gear icon ⚙️ next to the channel name
  2. Go to Integrations → Webhooks → New Webhook
  3. Name it (e.g. Instinct RPG), then click Copy Webhook URL and save it somewhere safe
  4. Click Save

You'll end up with two URLs that look like:
https://discord.com/api/webhooks/1234.../xxxx

⚠ Webhook URLs are only shown once. Copy both to a note before closing Discord.
Why this step
Without webhooks, there's no endpoint to post to.
Discord notifications work by Claude (or GitHub Actions) posting a JSON payload to a unique URL Discord gives you. Step 1 generates those URLs. Without them, Steps 2, 4, and the card rework skill can't fire any notifications — they'd have nowhere to send.
2
Add DISCORD_APP_WEBHOOK secret to GitHub Actions

This wires the #app-feedback webhook into the GitHub Action that's already live:

  1. Go to github.com/Nathan-Elequin/instinct-ttrpg → Settings → Secrets and variables → Actions
  2. Click New repository secret
  3. Name: DISCORD_APP_WEBHOOK
  4. Value: paste the #app-feedback webhook URL from Step 1
  5. Click Add secret
Test it: push any commit to main with the subject line starting with [Feature Update] — a Discord embed should appear in #app-feedback within ~30 seconds.
Why this step
The GitHub Action is live but blind — it has no URL to post to yet.
The discord-notify.yml file references secrets.DISCORD_APP_WEBHOOK. Without the secret set, the Action will run but the curl command will fail with a 401 or 404. Adding the secret gives it the destination. The secret stays out of the repo code, which is the right security pattern.
